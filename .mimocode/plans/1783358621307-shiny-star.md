# Автоматическое получение трендов — План

## Проблема

Сейчас `TrendScheduler` только истекает старые тренды. Тренды не берутся откуда-либо автоматически. `TrendResearchStage` использует хардкод "Interesting facts about technology" как fallback. Приложение не работает само по себе.

## Решение: 3 источника трендов

### 1. OpenAI — генерация трендовых идей (основной источник)

Уже есть OpenAI API. Добавляем метод `generateTrendTopics()` в `OpenAiService`:
- Промпт: "Придумай 10 трендовых тем для YouTube Shorts на русском языке на эту неделю. Формат JSON: {topics: [{topic, category, reason}]}"
- Стоимость: ~500 токенов, практически бесплатно
- Результат: 10 свежих тем каждые 6 часов

### 2. YouTube Data API v3 — поиск трендового контента

Уже есть YouTube API. Добавляем `YouTubeTrendService`:
- YouTube Search API: `search.list` с `order=viewCount`, `regionCode=RU`, `type=video`, `videoDuration=short`
- Парсим результаты: извлекаем query-слова из популярных видео
- Сохраняем как тренды с `source=youtube_trending`

### 3. OpenAI — анализ и обогащение трендов

Дополнительный промпт для проверки трендов:
- "Эта тема подходит для YouTube Shorts? Оцени от 1 до 10 по виральному потенциалу"
- Фильтруем темы с оценкой >= 7

### TrendScheduler — автоматизация

```java
@Scheduled(cron = "${scheduling.trend-cron:0 6 * * *}")
public void discoverTrends() {
    int newTrends = trendDiscoveryService.discoverTrends();
    log.info("Discovered {} new trends", newTrends);
    trendService.expireOldTrends();
}
```

### TrendResearchStage — убираем хардкод

```java
// Было:
trend = trendRepository.findFirstByStatusOrderBySearchVolumeDesc("new")
    .orElseGet(() -> {
        Trend manual = new Trend("Interesting facts about technology", "manual");
        return trendRepository.save(manual);
    });

// Стало:
trend = trendService.getBestTrend();
if (trend == null) {
    throw new PipelineStageException(name(), "No trends available. Run trend discovery first.");
}
```
